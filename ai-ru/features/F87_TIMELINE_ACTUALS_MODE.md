# F87 — Timeline: режим отображения факта (worklog / статусы Story)

**Версия:** 0.87.0
**Статус:** Реализовано (2026-07-06)
**Ветка:** `feat/f87-timeline-actuals-mode` (stacked поверх `feat/f86-quarterly-remaining-work`)

## Проблема

На вкладке Timeline прошлая (фактическая) часть баров сторей рисуется по залогированному
времени сабтасок (worklog-дни). Это показывает, *когда реально списывали время*, но не
показывает, *в каком статусе находилась стори*. Лид хочет уметь смотреть на факт двумя
способами и переключаться между ними.

## Решение

Переключатель (pill-dropdown) в панели фильтров Timeline рядом с «Week» и «Today (live)»:

| Режим | Что рисуем в прошлой части бара |
|-------|--------------------------------|
| **Logged time** (default) | Как сейчас: по-дневные сегменты worklog сабтасок, цвет = роль (SA/DEV/QA) |
| **Story statuses** | Интервалы из истории переходов статусов **самой Story** (`status_changelog`), цвет = цвет статуса из настроек (тот же, что у `StatusBadge` на Board) |

Переключатель влияет **только на прошлое**:
- retro-стори: весь бар по выбранному режиму;
- hybrid-стори: прошлая часть по выбранному режиму; будущая часть остаётся серой, когда
  прошлое нарисовано сегментами (worklog или статусы) — штриховка прогноз-фаз рисуется
  только при фазовом рендере (это существующее поведение worklog-режима, статусный режим
  его повторяет);
- forecast-стори: не затрагиваются.

Дефолт — «Logged time», выбор **не** сохраняется в localStorage (YAGNI).
Тултип бара не меняется.

## Backend

### 1. DTO: `RetrospectiveResult.RetroStory` + новое поле

```java
public record RetroStory(
        ...,
        List<WorklogDay> worklogDays,
        List<StatusInterval> statusIntervals   // NEW, всегда заполняется
) {}

/** Интервал нахождения стори в статусе (по её собственному status_changelog). */
public record StatusInterval(
        String status,
        LocalDate startDate,
        LocalDate endDate      // включительно
) {}
```

Поле добавочное — API обратно совместимо.

### 2. `RetrospectiveTimelineService`: построение интервалов

Changelog по ключам сторей уже загружается батчем
(`changelogRepository.findByIssueKeyInOrderByIssueKeyAscTransitionedAtAsc`) — новых запросов
в БД не требуется. Алгоритм по переходам одной стори (отсортированы по `transitionedAt`):

1. Первый интервал: статус = `fromStatus` первого перехода, начало = дата начала бара
   (`story.startDate`), конец = дата первого перехода.
2. Каждый переход `i` открывает интервал `toStatus[i]` с даты перехода `i` до даты
   перехода `i+1`.
3. Последний интервал: от последнего перехода до **сегодня**, если стори не завершена,
   иначе до даты последнего перехода (стори в Done — хвост не тянем).
4. Даты приводятся к `LocalDate`; интервалы нулевой длины (несколько переходов в один
   день) допустимы — фронт схлопнет их при отрисовке (последний статус дня выигрывает).
5. Нет ни одного перехода → `statusIntervals = []` (не null).

Никакого хардкода статусов: имена статусов передаются как есть, интерпретация (цвет) —
на фронте из конфигурации.

## Frontend

### 3. Переключатель

- `TimelinePage`: состояние `actualsMode: 'worklog' | 'status'`, default `'worklog'`.
- Pill-dropdown (тот же компонент, что «Week») с опциями `Logged time` / `Story statuses`
  добавляется в **обе** копии панели фильтров (обычную и sticky).
- Проп `actualsMode` прокидывается в `StoryBar`.

### 4. Отрисовка в `StoryBar`

- Режим `'worklog'` — текущее поведение без изменений.
- Режим `'status'` для retro/hybrid:
  - сегменты строятся из `statusIntervals`, клиппятся к границам бара
    (`startDate`..`endDate` бара) и к «сегодня» для hybrid;
  - цвет сегмента = `statusStyles[status]?.color`, fallback — нейтральный серый
    (прецедент прямого чтения цветов: `DsrBreakdownChart`);
  - `statusIntervals` пуст → сегменты не рисуются, остаётся серый фон бара;
  - будущая часть hybrid при сегментном рендере остаётся серой (паритет с worklog-режимом).
- Типы в `api/forecast.ts`: `RetroStory` + `statusIntervals: StatusInterval[]`.

## Тесты

**Backend (JUnit5, `RetrospectiveTimelineServiceTest`):**
- интервалы по цепочке переходов (To Do → In Progress → Testing → Done);
- незавершённая стори — последний интервал тянется до сегодня;
- завершённая стори — интервалы заканчиваются датой последнего перехода;
- стори без changelog — пустой список;
- несколько переходов в один день — интервалы не ломаются.

**Frontend (`TimelinePage.test.tsx`):**
- дефолт — worklog-рендер;
- переключение на «Story statuses» меняет отрисовку retro-бара на статусные сегменты;
- статус без цвета в конфиге — fallback-цвет, без падения.
