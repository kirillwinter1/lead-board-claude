# F6. Team Management UI

## Обзор

Две страницы для управления командами и участниками с настройками планирования.

## TeamsPage (`/teams`)

### Таблица команд
- Колонки: Название, Jira Team Value, Участники (количество), Дата создания, Действия
- Ссылка на страницу участников (`/teams/:id`)

### Управление
- **Создание/редактирование** — модальное окно (название + Jira Team Value)
- **Деактивация** — soft delete
- Кнопки управления показываются при `config.manualTeamManagement=true`

### Синхронизация
- Кнопка "Sync from Atlassian" (при наличии `organizationId`)
- Polling каждые 2 секунды во время синхронизации
- Отображение: время последней синхронизации, статус, ошибки

## TeamMembersPage (`/teams/:id`)

### Таблица участников
- Колонки: Имя, Jira Account ID, Роль (бейдж SA/DEV/QA), Грейд (бейдж), Часы/день, Действия
- Добавление/редактирование через модальное окно

### Модалка участника
- Jira Account ID (обязательно, заблокировано при редактировании)
- Display Name
- Роль: SA / DEV / QA (по умолчанию DEV)
- Грейд: Junior / Middle / Senior (по умолчанию Middle)
- Часы в день (по умолчанию 6.0)

### Planning Config (сворачиваемая секция)

**Коэффициенты грейдов:**
- Senior, Middle, Junior (min 0.1, max 3, step 0.1)
- Tooltip с объяснением множителей

**Risk Buffer:**
- Процент (0-100, step 5), рекомендация 15-25%

**Story Duration по ролям:**
- SA, DEV, QA (min 0.5, max 10, step 0.5)

**WIP Limits (рекомендательные):**
- Team, SA, DEV, QA (1-20)
- Авто-рекомендации на основе состава команды:
  - Team: участников / 3
  - По роли: количество × 1.5
- Индикаторы: зелёный (≤ рекомендации), жёлтый (≤ 150%), красный (> 150%)
- Предупреждение: "Не ограничивают планирование — Unified Planning использует реальный capacity"
