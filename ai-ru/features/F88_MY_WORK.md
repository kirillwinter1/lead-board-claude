# F88: My Work — личный рабочий стол участника команды

**Версия:** 0.88.0
**Статус:** В разработке
**Дата спеки:** 2026-07-07

## Проблема

Lead Board ориентирован на тимлида. Обычный участник команды (роль MEMBER) не имеет
причины заходить в продукт: его личная статистика существует (F30 Member Profile),
но спрятана во флоу тимлида (Teams → команда → участник) и построена как «страница
про кого-то», а не «про меня». Участнику нужно:

- видеть свои текущие и предстоящие задачи (в том числе горизонт «что мне скорее всего прилетит»);
- контролировать списание времени («не забыл ли я worklog»);
- понимать свою эффективность: соответствие списанного времени оценке (DSR), история
  закрытых задач, где именно промахиваюсь.

## Решение

Новая вкладка **My Work** — ежедневный рабочий стол участника. Наверху задачи и
контроль времени, аналитика эффективности ниже, вторым экраном.

Главный сценарий: участник заходит каждое утро → видит, что в работе и что дальше →
проверяет, всё ли время списано за неделю → раз в неделю листает вниз к самоанализу.

## Ключевые решения (из брейншторма)

| Вопрос | Решение |
|--------|---------|
| Главный сценарий | Рабочий стол на каждый день (задачи + worklog сверху, аналитика ниже) |
| Предстоящие задачи | Назначенные на меня + очередь команды по моей фазе |
| Worklog-виджет | Календарная сетка на 4 недели (текущая + 3 прошлых), только про себя |
| Доступ | Вкладка видна всем авторизованным; для роли MEMBER — стартовая страница |
| Мульти-команды | Агрегация по всем моим командам + опциональный фильтр-чип по команде |
| Аналитика | База из F30 + разрезы DSR по типу родителя и по эпикам |
| Подход | Новый `MyWorkService` + `GET /api/me/work`; F30 не ломаем |

## UI

Роут `/my-work`, вкладка «My Work» в `Layout.tsx` первой (перед Board). Язык UI —
английский (как после F61). Компоновка сверху вниз:

### 1. Шапка
Аватар, имя, роль/грейд, бейджи моих команд (`TeamBadge`), норма часов/день,
ближайшие мои отсутствия. Без редактирования компетенций (остаётся в F30).
Фильтр-чип по команде (если команд больше одной) — фильтрует блоки задач;
worklog и аналитика всегда персональные, по всем командам.

### 2. Сейчас в работе
Мои активные сабтаски из всех команд. Колонки: иконка типа (`getIssueIcon`),
ключ-ссылка в Jira, название, родительская Story и Epic, `StatusBadge`,
`TeamBadge`, оценка / уже списано (видно перерасход до закрытия задачи).

### 3. Предстоящие
Две секции:
- **Assigned to me** — назначенные на меня, ещё не начатые сабтаски;
- **Team queue** — ближайшие стори моих команд в порядке борда, где есть
  неназначенные сабтаски моей фазы. Фаза определяется из роли `TeamMember`
  через `WorkflowConfigService` (никакого хардкода SA/DEV/QA). Top-10.

### 4. Worklog-календарь (4 недели)
Сетка дней: текущая неделя + 3 прошлых. В каждом дне — списано vs норма
(`hoursPerDay` из `TeamMember`). Недосписанные рабочие дни подсвечены.
Отсутствия (из `AbsenceService`) и выходные исключены из подсветки.
Тултип по дню — разбивка списаний по задачам. Аналог `WorklogTimeline` (F65),
но для одного человека.

### 5. Моя эффективность (второй экран)
Период-селектор (from/to, как в F30). Содержимое:
- карточки `MetricCard`: Closed tasks, Avg DSR, Cycle Time, Hours fact/estimate;
- недельный тренд (DSR-линия + бары закрытых задач);
- таблица закрытых задач с DSR-бейджами (как в F30);
- **разрезы** — две компактные таблицы: DSR по типу родителя (Bug vs Story)
  и DSR по эпикам («где я промахиваюсь»).

### Empty-state
Если у пользователя нет связки `atlassianAccountId` ↔ `TeamMember` — вкладка
видна, страница показывает empty-state: «You are not a member of any team yet.
Ask your team lead to add you.»

### Стартовая страница для MEMBER
Индексный роут `/` редиректит на `/my-work`, если `auth.isMember()`.
Остальные роли попадают на Board, как раньше.

## Backend

### API
`GET /api/me/work?from=&to=&teamId=` — авторизованный пользователь, любая роль.
`teamId` опционален (фильтр-чип). 401 без сессии. Если нет `TeamMember` —
200 с пустым DTO и флагом `hasMembership: false` (фронт рисует empty-state).

### MyWorkService
Новый сервис. Поток: сессия → `atlassianAccountId` →
`TeamMemberRepository.findAllByJiraAccountIdAndActiveTrue` → агрегированный DTO:

```
MyWorkResponse {
  hasMembership: boolean
  member: { displayName, avatarUrl, role, grade, hoursPerDay, teams[] }
  upcomingAbsences[]
  activeTasks[]        // по всем командам (или отфильтровано teamId)
  upcomingAssigned[]
  teamQueue[]          // стори с неназначенными сабтасками моей фазы, top-10
  worklogCalendar[]    // 4 недели: day, loggedSeconds, normSeconds, isAbsence, isWeekend, byIssue[]
  analytics {
    summary { closedCount, avgDsr, cycleTimeDays, hoursSpent, hoursEstimated }
    weeklyTrend[]
    completedTasks[]
    dsrByParentType[]  // Bug vs Story (типы через WorkflowConfigService)
    dsrByEpic[]
  }
}
```

### Переиспользование (не копирование)
- Общая DSR-логика выносится из `MemberProfileService` в переиспользуемый
  компонент; `MemberProfileService` и `MyWorkService` используют его совместно.
- Данные: `JiraIssueRepository` (assignee-запросы уже есть),
  `IssueWorklogRepository.findDailyWorklogsByAuthors` (F57), `AbsenceService`,
  учёт пауз DSR из F32.
- Типы/статусы/фазы — только через `WorkflowConfigService`
  (`isStory()`, `isBug()`, `getSubtaskRole()`, `getRolesInPipelineOrder()`).
- Jira browse-URL — через `JiraConfigResolver`.

### Миграции
Не требуются: все данные уже в схеме (issues, worklogs, team_members, absences).

## Frontend

- `MyWorkPage.tsx` + css, роут `/my-work` в `App.tsx`, `NavLink` в `Layout.tsx`.
- Таблица задач и DSR-бейдж выносятся из `MemberProfilePage` в shared-компоненты
  (`components/member/`), используются в обоих местах.
- Дизайн-система: `getIssueIcon`, `StatusBadge`, `TeamBadge`, `MetricCard` — как есть.
- API-клиент: `frontend/src/api/myWork.ts`.

## Тесты

- **JUnit (MyWorkService):** агрегация по нескольким командам; фазовая очередь
  (фильтр по роли участника, порядок борда, top-10, исключение назначенных);
  worklog-календарь (норма, отсутствия, выходные); разрезы по типу/эпику;
  `hasMembership=false` без TeamMember.
- **JUnit (controller):** 401 без сессии; 200 с данными; фильтр teamId.
- **Vitest:** страница рендерит блоки; empty-state; редирект MEMBER с `/`.

## Вне скоупа (бэклог)

- Разбор промахов «почему вышел за оценку» (контекст пауз, комментарии).
- Напоминания о недосписанном времени (нотификации, BF7).
- Списание времени со страницы (Jira — источник истины; пока только ссылки).
- Урезание остальной навигации для роли MEMBER.
