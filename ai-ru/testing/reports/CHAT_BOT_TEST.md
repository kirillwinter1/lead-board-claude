## Паттерны ошибок (204 вопроса, 20.4%)

### 1. Не знает команду пользователя — 40 вопросов

**Примеры:**
- Q75: «В какой я команде?» → «Не могу определить»
- Q86: «Покажи мою команду» → «Не могу определить»
- Q97: «Как дела у моей команды?» → «Укажите ID команды»
- Q111: «Какой DSR у моей команды?» → «Уточните ID»

**Причина:** System prompt содержит `Команды пользователя: все (ADMIN имеет доступ ко всем командам)`, но бот не знает, к какой команде пользователь «привязан» по умолчанию. Для ADMIN нет «моей» команды.

**Решение:** Для не-ADMIN пользователей добавить в контекст `primary_team_id`. Для ADMIN — отвечать по всем командам.

### 2. Не может найти команду по названию — 27 вопросов

**Примеры:**
- Q87: «Сколько задач у команды Backend?» → «Нет информации о ID»
- Q89: «Какой throughput у Frontend?» → «Не могу по названию»
- Q91: «Сколько багов у QA?» → «Нет данных о команде»
- Q200: «Покажи данные для команды Backend» → «Не могу найти»

**Причина:** Все tools принимают только `teamId: integer`. Бот получает в system prompt список команд с ID, но LLM не всегда может сматчить произвольное название с конкретным ID.

**Решение:** Добавить параметр `teamName: string` в tools, резолвить на бэкенде. Или добавить tool `team_resolve(name) → id`.

## Вывод

### Сильные стороны
- **0 HTTP ошибок** — стабильная работа на 1000 запросах
- **Точная KB** — когда бот отвечает из knowledge base, информация обычно верная
- **Tool usage корректный** — когда tools вызываются, данные точные
- **Длинные ответы на сложные вопросы** — стратегические и аналитические вопросы получают развёрнутые ответы

### Слабые стороны
- **20.4% вопросов без ответа** — «не могу», «нет данных», «уточните»
- **Только 4.3% tool usage** — бот слишком редко обращается к реальным данным
- **Проблема «моя команда»** — 40 вопросов не работают из-за невозможности определить команду пользователя
- **Проблема «команда по имени»** — 27 вопросов не работают из-за отсутствия резолвинга имени → ID
- **Галлюцинации tool names** — 1 случай, но потенциально опасно
- **~15 неправильных ответов** — серьёзная проблема качества

### Приоритет фиксов

**Quick wins (D1–D3, D8, D9, D10):** Изменения в system prompt + резолвинг команд.
Эффект: убирает ~120 из 204 failures (~60%).

**Medium effort (D4–D7):** Новые tools.
Эффект: покрывает ещё ~110 вопросов.

**Итого после всех фиксов:** ~90%+ вопросов будут получать содержательный ответ.
