## Scope
Новая категория статуса `DEV_DONE` в пайплайне эпиков:
`NEW → REQUIREMENTS → PLANNED → IN_PROGRESS → DEV_DONE → DONE`

Цель: DSR перестаёт считать дни когда эпик переходит в DEV_DONE (E2E/UAT/приёмка), но эпик не помечается как завершённый.

## Bugs Found

### BUG-135 (High): DataQualityService — epicPastTodo не включает DEV_DONE
**Файл:** `DataQualityService.java:75-78`
**Описание:** Переменная `epicPastTodo` проверяет PLANNED, IN_PROGRESS, DONE, но пропускает DEV_DONE.
**Ожидаемый результат:** Эпик в DEV_DONE считается "past todo" и проходит все data quality проверки.
**Фактический результат:** Эпик в DEV_DONE обрабатывается как "still planning" — могут ложно срабатывать EPIC_NO_ESTIMATE и другие правила.
**Fix:**
```java
boolean epicPastTodo = epicCategory == StatusCategory.PLANNED
        || epicCategory == StatusCategory.IN_PROGRESS
        || epicCategory == StatusCategory.DEV_DONE
        || epicCategory == StatusCategory.DONE;
```

### BUG-136 (High): MemberProfileService — DEV_DONE задачи исчезают из view
**Файл:** `MemberProfileService.java:82-87`
**Описание:** Active tasks включают только IN_PROGRESS и PLANNED. Upcoming — только isNotStarted(). DEV_DONE не попадает ни в одну группу.
**Ожидаемый результат:** Задачи в DEV_DONE отображаются в "Active Tasks".
**Фактический результат:** Задачи в DEV_DONE пропадают из профиля участника.
**Fix:**
```java
if (statusCat == StatusCategory.IN_PROGRESS || statusCat == StatusCategory.PLANNED || statusCat == StatusCategory.DEV_DONE) {
    activeTasks.add(buildActiveTask(issue, issueCache));
}
```

### BUG-137 (Medium): getFirstStatusNameForCategory — нет fallback для DEV_DONE
**Файл:** `WorkflowConfigService.java:664-669`
**Описание:** Hardcoded fallback для status names не включает DEV_DONE. При отсутствии маппинга в БД возвращает null.
**Ожидаемый результат:** Fallback возвращает осмысленное имя (e.g. "Dev Done").
**Фактический результат:** Возвращает null (через default → null).
**Fix:**
```java
return switch (target) {
    case DONE -> "Done";
    case IN_PROGRESS -> "In Progress";
    case DEV_DONE -> "Dev Done";
    case NEW -> "New";
    default -> null;
};
```
