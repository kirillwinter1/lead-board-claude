## Bugs Found

### Critical (2)

| Bug ID | Description | File | Status |
|--------|------------|------|--------|
| BUG-94 | **AuthorizationService uses getRole() (global) instead of getTenantRole().** All `@PreAuthorize("@authorizationService.canManageTeam()")` etc. checks compare against user's global `app_role`, not per-tenant role. A global ADMIN bypasses tenant-level restrictions; a tenant ADMIN with global MEMBER role is denied access. Spring's `hasRole()` works correctly (uses `getAuthorities()` from tenant role), but SpEL `@authorizationService.*` methods don't. | `AuthorizationService.java:48,56,64,75,89,93,108,119` | OPEN |
| BUG-95 | **Cross-tenant session reuse.** Session cookie from tenant A is accepted on tenant B. `LeadBoardAuthenticationFilter` never validates that `session.tenantId` matches `TenantContext.getCurrentTenantId()`. If user has no `tenant_users` record in tenant B, fallback gives them global role (which could be ADMIN). Cookie domain may be `.leadboard.app` (shared). | `LeadBoardAuthenticationFilter.java:46-66` | OPEN |

### High (5)

| Bug ID | Description | File | Status |
|--------|------------|------|--------|
| BUG-96 | **First-tenant-user ADMIN logic is broken.** Every user who authenticates against a tenant they haven't joined yet gets ADMIN. The check `isFirstTenantUser` verifies if *this user* is in *this tenant*, NOT if the tenant has any users at all. Any user + any tenant = ADMIN. | `OAuthService.java:153-161` | OPEN |
| BUG-97 | **11+ write endpoints lack @PreAuthorize.** Competency PUT, Epic PATCH (rough estimate), Forecast snapshot POST, AutoScore recalculate POST, Planning recalculate POST, Calendar refresh POST, WIP snapshot POST, AuditRequest POST, Poker (all CRUD) — all accessible to any authenticated user including VIEWER. | Multiple controllers | OPEN |
| BUG-98 | **WebSocket endpoint has no authentication.** `/ws/**` is `permitAll()` in SecurityConfig. No handshake interceptor validates session cookie. Any unauthenticated user can connect to `/ws/poker/{roomCode}`, observe votes, and potentially submit votes. | `SecurityConfig.java:59`, `PokerWebSocketConfig.java` | OPEN |
| BUG-99 | **findLatestToken() returns any user's OAuth token.** All Jira API calls use whichever user's token was most recently refreshed, regardless of who is making the request. Privilege confusion — API calls use wrong user's Jira identity/permissions. | `OAuthService.java:208-224`, `OAuthTokenRepository.java:14-15` | OPEN |
| BUG-100 | **No session invalidation on role change.** When admin changes a user's role, existing sessions continue with old role. If user is removed from `tenant_users`, fallback to global role gives elevated access. Sessions last 30 days by default. | `LeadBoardAuthenticationFilter.java:54` | OPEN |

## Recommendations

### P0 (Critical — security)
1. **Fix AuthorizationService to use getTenantRole()** — all `auth.getRole()` → `auth.getTenantRole()`
2. **Validate session tenant binding** — reject session if `session.tenantId != TenantContext.currentTenantId`
3. **Fix first-tenant-user ADMIN logic** — check if tenant has ANY users, not just this user

### P1 (High)
4. **Add @PreAuthorize to 11+ unprotected write endpoints** — at minimum `hasAnyRole('ADMIN', 'TEAM_LEAD')` for competency, forecast, autoscore, planning, calendar write operations; VIEWER should be read-only
5. **Add WebSocket authentication** — handshake interceptor to validate session cookie
6. **Scope OAuth token to current user** — change `findLatestToken()` to `findByUserId(userId)`
7. **Invalidate sessions on role change** — delete all sessions for user when role updated

### P2 (Medium)
8. **Add RBAC integration tests** — test that MEMBER/VIEWER are rejected from admin endpoints
9. **Add tenant registration rate limiting** — per-IP throttle or CAPTCHA
10. **Move OAuth state to database or Redis** — needed for multi-instance deployments
11. **Add frontend route guards** — redirect non-admins from `/board/settings` to 403 page
12. **Lock first-user ADMIN check** — advisory lock or SELECT FOR UPDATE

### P3 (Low)
13. **Encrypt OAuth tokens at rest** — encrypt access_token/refresh_token columns
14. **Add session fingerprinting** — optional IP/User-Agent binding
