## Bugs Found

### High

**BUG-49: Unified/Forecast 500 на несуществующей команде (teamId=9999)**
- **Endpoint:** `GET /api/planning/unified?teamId=9999`, `GET /api/planning/forecast?teamId=9999`, `GET /api/planning/role-load?teamId=9999`
- **Ожидаемое:** 404 или 200 с пустым результатом (как retrospective)
- **Фактическое:** 500 Internal Server Error
- **Причина:** UnifiedPlanningService/ForecastService бросает TeamNotFoundException, не перехватывается @ExceptionHandler
- **Заметка:** `GET /api/planning/retrospective?teamId=9999` корректно возвращает 200 с пустым списком — поведение несогласовано между endpoints
- **Статус:** ✅ FIXED (GlobalExceptionHandler + TeamNotFoundException → 404)

**BUG-50: WIP Status в ForecastResponse — лимиты не из конфига команды**
- **Endpoint:** `GET /api/planning/forecast?teamId=1`
- **Ожидаемое:** WIP limits из team planning_config (SA=2, DEV=3, QA=2, team=7)
- **Фактическое:** Все role limits = количеству эпиков (6), team limit = 6
- **Причина:** `ForecastService.java:95` — `RoleWipStatus.of(forecasts.size(), forecasts.size())` — хардкодит размер списка вместо конфига
- **Влияние:** WIP никогда не показывает exceeded, информация бесполезна
- **Статус:** ✅ FIXED (WIP limits из PlanningConfigDto.wipLimits)

**BUG-51: TimelinePage.test.tsx — 6 тестов падают после F45 (missing getRetrospective mock)**
- **Файл:** `frontend/src/pages/TimelinePage.test.tsx:15-21`
- **Причина:** F45 добавил вызов `getRetrospective(selectedTeamId)` в TimelinePage, но mock в тестах не включает эту функцию
- **Фактическое:** `TypeError: getRetrospective is not a function`
- **Fix:** Добавить `getRetrospective: vi.fn()` в mock и `getStatusStyles` в config mock
- **Статус:** ✅ FIXED (mocks added + WorkflowConfigContext mock)

**BUG-52: Race condition — нет AbortController при смене команды**
- **Файл:** `frontend/src/pages/TimelinePage.tsx:1469-1534`
- **Причина:** `useEffect` загружает 3 API параллельно (`Promise.all`), но при смене команды старый запрос не отменяется. Быстрое переключение команд может показать данные предыдущей команды
- **Cleanup:** Есть cleanup для `animationTimeoutRef`, но не для HTTP-запросов
- **Статус:** ✅ FIXED (AbortController + signal.aborted check)

### Medium

**BUG-53: WIP history принимает отрицательные days (-5) — inverted date range**
- **Endpoint:** `GET /api/planning/wip-history?teamId=1&days=-5`
- **Ожидаемое:** 400 Bad Request
- **Фактическое:** 200 OK, `from: "2026-03-02", to: "2026-02-25"` (from > to)
- **Причина:** Нет валидации параметра `days`
- **Статус:** ✅ FIXED (days > 0 validation → 400)

**BUG-54: Story Forecast — assigneeDisplayName=null для завершённых сторей**
- **Endpoint:** `GET /api/planning/epics/LB-95/story-forecast?teamId=1`
- **Фактическое:** LB-100, LB-108 имеют `assigneeDisplayName: null`, `isUnassigned: true`, `workDays: 0`
- **Причина:** Стори полностью залогированы (workDays=0), но displayName не заполняется для completed stories
- **Влияние:** На UI assignee показывается пустым
- **Статус:** ✅ FIXED (fallback displayName из assigneeSchedule)

**BUG-55: Historical snapshot mode не загружает retro данные**
- **Файл:** `frontend/src/pages/TimelinePage.tsx:1489-1504`
- **Причина:** При выборе исторической даты загружаются только `getForecastSnapshot` и `getUnifiedPlanningSnapshot`, но не retrospective. Функция `mergeHybridEpics()` не вызывается для исторического режима
- **Влияние:** Исторические снэпшоты не показывают гибридную визуализацию (retro + forecast merge)
- **Статус:** ✅ FIXED (historical mode loads retro + mergeHybridEpics)

**BUG-56: mergeHybridEpics() — потеря retro-only эпиков**
- **Файл:** `frontend/src/pages/TimelinePage.tsx:1222`
- **Причина:** Функция итерирует только `planEpics.map()`. Если эпик есть в retrospective, но НЕТ в unified plan (например, удалён или перенесён), его данные теряются
- **Влияние:** Исторические данные по удалённым/перенесённым эпикам не видны на таймлайне
- **Статус:** ✅ FIXED (retro-only epics added to merge result)

**BUG-57: Silent error catch при загрузке StatusStyles и Config**
- **Файл:** `frontend/src/pages/TimelinePage.tsx:1436-1438`
- **Причина:** `getStatusStyles().catch(() => {})` — ошибка полностью проглатывается. `getConfig().catch(console.error)` — только в console
- **Влияние:** Если StatusStyles не загрузились, статусы показываются серыми без индикации ошибки. Если Config не загрузился, Jira ссылки сломаны
- **Статус:** ✅ FIXED (console.error вместо silent catch)

### Low

**BUG-58: Tooltip может выйти за границы экрана**
- **Файл:** `frontend/src/pages/TimelinePage.tsx:386-389`
- **Причина:** `setTooltipPos({ x: rect.right + 8, y: rect.top })` — нет проверки на overflow right/bottom
- **Влияние:** При наведении на элементы у правого края экрана tooltip обрезается
- **Статус:** ✅ FIXED (viewport boundary clamping)

**BUG-59: Нет aria-labels на интерактивных элементах Gantt**
- **Файл:** `frontend/src/pages/TimelinePage.tsx` — select (1625), story-bar click handlers (717), gantt-chart container (1732)
- **Причина:** `<select>` без aria-label, story bars без role="button", gantt без role="region"
- **Влияние:** Screen reader пользователи не могут навигировать по таймлайну
- **Статус:** ✅ FIXED (aria-label на selects, role=button на story-bars, role=region на gantt)
