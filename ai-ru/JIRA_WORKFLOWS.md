# Jira Workflows

Документация по workflow для разных типов задач в Jira.

## Epic Workflow

```
НОВОЕ → REQUIREMENTS → ROUGH ESTIMATE → ЗАПЛАНИРОВАНО → DEVELOPING → E2E TESTING → ACCEPTANCE → ГОТОВО
```

| Статус | Этап | Описание | AutoScore |
|--------|------|----------|-----------|
| **НОВОЕ** | Подготовка | Эпик только создан, работа не начата | -5 |
| **REQUIREMENTS** | Подготовка | Пишутся бизнес-требования (БТ) | +5 |
| **ROUGH ESTIMATE** | Подготовка | Грубая оценка трудозатрат по ролям | +10 |
| **ЗАПЛАНИРОВАНО** | Подготовка | Эпик готов к разработке, ожидает очереди | +15 |
| **DEVELOPING** | Исполнение | Активная разработка (стори в работе) | +25 |
| **E2E TESTING** | Исполнение | End-to-end тестирование всего эпика | +30 |
| **ACCEPTANCE** | Исполнение | Приёмка заказчиком | +30 |
| **ГОТОВО** | Завершён | Эпик закрыт | N/A (не отображается) |

### Принцип приоритизации эпиков

1. **Закончить начатое** — эпики на поздних стадиях workflow имеют высший приоритет
2. **Jira Priority** — бизнес-приоритет из Jira имеет значительный вес
3. **Due Date** — приближающийся дедлайн повышает приоритет
4. **Размер** — меньшие эпики получают небольшой бонус (быстрее закрыть)
5. **Возраст** — старые эпики получают небольшой бонус

## Story Workflow

```
НОВОЕ → ANALYSIS → ANALYSIS REVIEW → WAITING DEV → DEVELOPMENT → DEV REVIEW → WAITING QA → ТЕСТИРОВАНИЕ → TEST REVIEW → READY TO RELEASE → ГОТОВО
```

| Статус | Фаза | Роль | Описание |
|--------|------|------|----------|
| **НОВОЕ** | - | - | Стори создана, работа не начата |
| **ANALYSIS** | SA | Аналитик | Написание технических требований |
| **ANALYSIS REVIEW** | SA | Аналитик | Ревью требований |
| **WAITING DEV** | - | - | Ожидание разработчика |
| **DEVELOPMENT** | DEV | Разработчик | Активная разработка |
| **DEV REVIEW** | DEV | Разработчик | Code review |
| **WAITING QA** | - | - | Ожидание тестировщика |
| **ТЕСТИРОВАНИЕ** | QA | Тестировщик | Тестирование |
| **TEST REVIEW** | QA | Тестировщик | Ревью тестов / регресс |
| **READY TO RELEASE** | - | - | Готово к релизу |
| **ГОТОВО** | - | - | Стори закрыта |

### Маппинг статусов на фазы (для планирования)

| Фаза | Статусы |
|------|---------|
| **SA** (Analytics) | ANALYSIS, ANALYSIS REVIEW |
| **DEV** (Development) | WAITING DEV, DEVELOPMENT, DEV REVIEW |
| **QA** (Testing) | WAITING QA, ТЕСТИРОВАНИЕ, TEST REVIEW |
| **Done** | READY TO RELEASE, ГОТОВО |

## Sub-task Workflow

```
НОВОЕ → В РАБОТЕ → ПРОВЕРКА → ГОТОВО
```

| Статус | Описание |
|--------|----------|
| **НОВОЕ** | Подзадача создана |
| **В РАБОТЕ** | В процессе выполнения |
| **ПРОВЕРКА** | На проверке (ревью) |
| **ГОТОВО** | Подзадача закрыта |

## Использование в Lead Board

### AutoScore для эпиков

AutoScore определяет порядок эпиков на доске и в планировщике.

**Факторы и веса:**

| Фактор | Макс. баллов | Описание |
|--------|--------------|----------|
| Status | -5 to +30 | Позиция в workflow |
| Jira Priority | 0-20 | Highest=20, High=15, Medium=10, Low=5 |
| Due Date | 0-25 | Экспоненциальный рост к дедлайну |
| Progress | 0-10 | % выполнения (logged/estimate) |
| Size | -5 to +5 | Меньше = выше (без оценки = -5) |
| Age | 0-5 | Логарифм от возраста |
| Manual Boost | ±∞ | Ручная корректировка |

### AutoScore для сторей

Определяет порядок сторей внутри эпика для планирования.

**Факторы:**
- Статус в workflow (дальше = выше)
- Flagged (блокер = +10)
- Blocked by (заблокирована = -5)
- Jira Priority
- Возраст
