# Главная страница Lead Board

Главная страница — это **единая доска управления поставкой** для выбранной компании и Jira-проекта.

Она отвечает на главный вопрос руководителя:

> **Что сейчас делает команда, сколько это стоит по времени и когда мы закончим?**

---

## 1. Структура данных

Доска отображает иерархию задач, синхронизированную с Jira:

* **Epic** — крупная бизнес-инициатива
* **Story** — часть эпика
* **Sub-task** — аналитика, разработка или тестирование

Иерархия:

```
Epic
 └─ Story
     ├─ Analytics sub-task
     ├─ Development sub-task
     └─ Testing sub-task
```

Все данные по задачам, статусам и времени берутся из Jira,
расчёты выполняются на стороне Lead Board.

---

## 2. Что отображается на доске

Для каждого **Epic** на доске показывается:

| Поле | Описание |
|------|----------|
| Ключ задачи | `LB-123`, кликабелен → ведёт в Jira |
| Название | Краткое описание эпика |
| Статус | Текущий статус из Jira |
| Команда | Назначенная команда |
| Грязная оценка | В человеко-днях (если эпик в статусе планирования) |
| Суммарная оценка | После декомпозиции (из sub-tasks) |
| Фактическое время | Залогированное время |
| Прогресс | Общий и по ролям |

Для **Story** и **Sub-task** отображаются соответствующие оценки и прогресс.

---

## 3. Прогресс и расчёты

Lead Board считает:

* Оценки и факт **по ролям**:
  * Аналитика (SA)
  * Разработка (DEV)
  * Тестирование (QA)
* Общий прогресс эпика
* Прогресс не может превышать 100%
* Если факт превышает оценку — это подсвечивается как проблема качества данных

Все расчёты выполняются на бэке, фронт только отображает результат.

---

## 4. Грязные оценки (планирование)

Если Epic находится в разрешённом статусе (например, Backlog):

* Можно внести **грязную оценку** в человеко-днях
* Шаг — 0,1
* Используется для:
  * Квартального планирования
  * Расчёта capacity команды
* После перехода эпика в работу оценка становится read-only

Разрешённые статусы настраиваются гибко для каждого Jira-проекта.

---

## 5. Фильтры и поиск

На главной странице доступны:

| Фильтр | Описание |
|--------|----------|
| Поиск | По ключу или названию задачи |
| Команды | Фильтр по одной или нескольким командам |
| Статусы | Фильтр по статусам задач |
| Чекбокс | «Показывать задачи других команд» |
| Пагинация | Ленивый скролл / курсорная пагинация |

Фильтрация и пагинация работают через backend.

---

## 6. Data Quality (качество данных)

Доска подсвечивает проблемы в данных:

| Проблема | Описание |
|----------|----------|
| Нет команды | У эпика не назначена команда |
| Нет оценки | Есть залогированное время, но нет оценки |
| Нет декомпозиции | Эпик не декомпозирован, но в работе |
| Превышение | Факт превышает оценку |

Предупреждения показываются через иконки и ховеры.
В будущем будет отдельная вкладка **Data Quality**.

---

## 7. Целевая аудитория

Главная страница предназначена для:

* Тимлидов
* Delivery-менеджеров
* Руководителей разработки
* Product / Project менеджеров

> Это **не операционная доска разработчика**,
> а инструмент управления поставкой и прогнозированием.
