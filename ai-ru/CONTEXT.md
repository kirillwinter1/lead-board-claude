# Контекст и видение продукта

## Проблема

IT-команды и менеджмент сталкиваются с вопросами:
- Сколько работы у нас на самом деле?
- Какой реальный прогресс?
- Успеем ли в срок?
- Где риски доставки?

Jira даёт данные, но не даёт ответов. Нужен инструмент для анализа и прогнозирования.

---

## Решение

Lead Board — надстройка над Jira, которая:
- Агрегирует данные по иерархии (Epic → Story → Sub-task)
- Считает прогресс по ролям (Analytics, Development, Testing)
- Показывает capacity команд
- Выявляет проблемы качества данных
- Помогает планировать доставку

---

## Принципы продукта

### 1. Прозрачность важнее оптимизма
Показываем реальную картину, даже если она неприятная.
Лучше узнать о проблеме сейчас, чем удивиться в конце квартала.

### 2. Jira — источник истины
Не дублируем Jira, не заменяем её.
Все задачи, статусы, worklogs живут в Jira.
Lead Board только читает и анализирует.

### 3. Конфигурация вместо кастомизации
Разные команды используют Jira по-разному.
Система должна адаптироваться через конфигурацию, а не через код.

### 4. Факты вместо обещаний
Планирование на основе исторических данных.
Capacity — это факт, а не пожелание.

---

## Целевые пользователи

### Team Lead
- Видит прогресс своей команды
- Управляет составом команды (роли, грейды, часы)
- Получает алерты о проблемах данных

### Engineering Manager
- Видит все команды в своём scope
- Сравнивает прогресс между командами
- Планирует capacity на квартал

### Product Manager
- Видит прогресс по Epic'ам
- Понимает, какие фичи в риске
- Принимает решения о приоритетах

### Delivery Manager
- Видит общую картину
- Отслеживает delivery metrics
- Готовит отчёты для stakeholders

---

## Что Lead Board НЕ делает

- Не заменяет Jira
- Не модифицирует данные в Jira
- Не является task tracker'ом
- Не является HR-системой
- Не даёт оценки автоматически

---

## Ключевые сценарии

### Сценарий 1: Ежедневный статус
Team Lead открывает Board, видит:
- Прогресс по всем Epic'ам команды
- Алерты о проблемах (done без логов, оценка 0)
- Capacity команды на неделю

### Сценарий 2: Планирование квартала
Engineering Manager:
- Смотрит dirty estimates для новых Epic'ов
- Сравнивает с доступным capacity
- Принимает решение о scope

### Сценарий 3: Ретроспектива
Team Lead анализирует:
- Сколько времени занял Epic
- Как оценки соотносились с реальностью
- Где были недооценки/переоценки

---

## Метрики качества данных

Lead Board помогает поддерживать гигиену Jira:

| Проблема | Алерт |
|----------|-------|
| Epic без команды | Team field not set |
| Story без estimate | No estimate |
| Done без логов | Done but no time logged |
| Логи > оценки | Overlogged |
| Sub-task без роли | Unknown sub-task type |

## Roadmap (высокоуровневый)

### MVP (текущий фокус)
- Board с Epic → Story → Sub-task
- Синхронизация из Jira
- OAuth через Atlassian
- Базовое управление командами

### v1.0
- Multi-tenancy
- Гибкая конфигурация per JiraSpace
- Полный RBAC
- Dirty estimates для планирования

### v2.0 (будущее)
- Capacity planning
- Auto-planning suggestions
- Historical analytics
- Predictive delivery dates
